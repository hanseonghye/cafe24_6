[용어 정리]

IP 주소의 종류 = 글로벌 / 사설
서브넷 마스크 = 연속하는 IP 주소 그룹핑 for 한정된 네트워크 자원 공유
도메인 = IP 주소를 식별하기 쉽게 해주는 문자열 형태의 주소
DNS = 도메인 및 IP 주소 간 변환을 용이하게 해주는 분산형 DB 시스템
www(서브 도메인).example(사명).co(속성).kr(국가코드)
라우터 = 상위 및 하위 네트워크 간 연결
NAT = 사설 IP 주소를 활용하여 글로벌 네트워크와 통신할 수 있게 해주는 기술
URI = 넷 상에서의 데이터 위치를 나타내기 위한 표현 방법(URL 포함)
http(프로토콜)://example.com(도메인 주소)/(경로)
프로토콜별 대표적인 포트번호가 대략적으로 정해져 있음 e.g) SMTP(25)
방화벽 = 로컬과 인터넷의 사이 위치하며 둘 사이 통신 제어
SSL = 통신의 암호화
서버 스펙을 결정하기 어렵다면 클라우드 시스템 대체 활용 권고
서버 디스크 결정 시 디스크 자체보다 적절한 RAID 카드 선택이 더 중요
RAID = 데이터의 분산 및 다중화 수행
데이터 베이스 성능 관련 키워드 ACID (원자성 / 일관성 / 독립성 / 지속성)
락과 배타처리 = 어떤 처리가 리소스 사용 시 다른 처리가 해당 리소스 사용 불가
[처리 효율화 및 보틀넥 완화]
1) 버퍼 = 다음 단계 처리 위해 데이터 일시 저장
2) 캐시 = 처리 결과 일시적을 모아두며 재사용
3) 큐잉 = 처리큐 + 결과큐로 구성 / 비동기성이 핵심
다중화의 경우 같은 데이터가 여러개 존재하므로 어떤 것이 가장 최신의 것인지 주의해야 함
[스토리지 공유 방식]
1) shared disk = 하나의 스토리지 공유 / 정합성 문제 없음
2) shared nothing = 여러 스토리지 간 통신을 통해 정합성 유지 (리플리케이션)
리플리케이션 = 송신(마스터) / 수신(슬레이브)
MySQL and PostgreSQL = 리플리케이션 적용
페일오버 = 대기 -> 활성화 단계
스플릿 브레인 = 마스터가 다운되지 않았는데 슬레이브가 페일오버 되는 현상
스토니스 = 스플릿 브레인과 같은 이상현상 발생 시 강제로 마스터 종료시키는 동작
암호화 = 원 데이터 -> 암호
복호화 = 암호 -> 원 데이터 (가역 과정)
[암호화 방식]
1) 공통키 = 패스워드 거는 방식
2) 공개키 = 암호화 및 복호화에 각각 다른 키를 준비하여 암호화 쪽만 공개
해시의 경우 원래대로 되돌릴 수 없는 불가역적 특성 지님
salt = 변환 과정에서 해독의 난이도를 높이기 위해 추가하는 특정 문자열
